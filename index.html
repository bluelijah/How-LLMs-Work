01 - Tokenization

Text gets broken
into tokens

Before anything else, your text is split into subword chunks called tokens. The model never sees raw characters - it sees a sequence of token IDs, each representing a token from a fixed vocabulary of ~50,000 entries.

02 - Embedding

Token IDs become
dense vectors

Each token ID is looked up in a giant table to produce a vector of floats, typically thousands of dimensions. A vector is a list of values, in this instance, decimal precise values (0.1, -0.5, 0.8) called floats. Each dimension is one of these float values. Semantically similar words land near each other. This is where meaning first enters the model.

03 - Architecture

Stacked transformer
layers

The embedded vectors pass through N transformer layers, each refining the representation. Hover any layer to see what it does.

Input tokens
Raw text → token IDs
Your prompt as a sequence of integers. "The cat" might be [464, 3797].
Embedding + positional encoding
IDs → vectors with position info
Token IDs become dense vectors. A sine/cosine positional signal is added so the model knows token order.
Transformer block x N
Self-attention + feed-forward + layer norm
x 32 to 128 layers
Each block runs attention (tokens look at each other) then a feed-forward network. This is where reasoning happens.
Output projection
Final vector → vocab logits
The last token's vector gets multiplied by a weight matrix to produce one score per vocabulary entry.
Softmax → sample
Scores → probabilities → next token
Softmax converts raw scores into probabilities summing to 1. The model samples from this distribution to pick the next token.

05 - Generation

One token

The model generates text autoregressively - it produces one token, appends it to the input, then runs the whole forward pass again. Every token is a fresh prediction.

06 - Training

Learning from
prediction error

The model trains by repeatedly predicting the next token, measuring how wrong it was, and nudging billions of weights to do better. This loop runs across trillions of tokens.

01
Forward pass
Input runs through every layer. Each token gets a probability distribution over the vocabulary.
02
Loss calculation
Cross-entropy loss measures how low a probability was assigned to the actual next token.
03
Backpropagation
The chain rule computes each weight's contribution to the error - billions of gradients.
04
Weight update
Adam optimizer adjusts every weight slightly in the direction that reduces loss.

07 - Q, K, V

The mechanics
of attention

Every token produces three vectors via learned weight matrices. Their interaction determines how meaning flows across the sequence.

1
Produce Q, K, V
Q = x·Wq   K = x·Wk   V = x·Wv
Each token's embedding x is multiplied by three separate learned matrices. Q asks "what am I looking for?", K says "what do I offer?", V says "here's what I'll contribute."
2
Compute attention scores
score(i,j) = Qᵢ · Kⱼ / √d
The dot product of token i's Query with token j's Key gives a raw score. Dividing by √d (dimension) keeps gradients stable.
3
Normalize with softmax
αᵢⱼ = softmax(scoreᵢⱼ)
Softmax turns the raw scores into weights that sum to 1 - a probability distribution over how much to attend to each token.
4
Weighted sum of Values
outputᵢ = Σⱼ αᵢⱼ · Vⱼ
The output for token i is a blend of every token's V, weighted by the attention scores. "bank" borrows heavily from "river" - its output vector is now contextually disambiguated.
08 - RLHF

Making it
helpful

A pretrained model is good at predicting text, not following instructions. Reinforcement Learning from Human Feedback fine-tunes it to be genuinely useful - helpful, honest, and safe.

Pretrained base model
Knows language deeply. Will complete any text including harmful ones.
Supervised fine-tuning
Train on human-written ideal responses to build an instruction-following baseline.
Reward model training
Humans rank model outputs. A separate reward model learns to predict human preference.
RL fine-tuning (PPO)
The policy model is updated to maximize the reward signal - outputs humans prefer score higher.
Deployed assistant
A model that generates text that is useful, safe, and aligned with human intent.
